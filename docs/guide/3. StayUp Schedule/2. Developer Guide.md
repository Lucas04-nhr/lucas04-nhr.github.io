This project is a standard Flutter multi-platform application (Android / iOS / macOS / Windows / Linux / Web). The following content is adapted from the StayUP-Calendar repository README.

## 1. Development Environment

- Flutter SDK: It is recommended to use the latest stable version (must be compatible with `Dart >=3.11`, see `pubspec.yaml`).
- Xcode: Used for iOS/macOS debugging and packaging (required for macOS development).
- Android Studio or Android SDK Command-line Tools: Used for Android debugging and packaging.
- Optional: Chrome (Web debugging), Visual Studio Build Tools (Windows builds).

After the first clone, run:

```bash
flutter doctor
flutter pub get
```

## 2. Project Structure

- `lib/main.dart`: Main business logic and UI entry point (core logic is currently concentrated here).
- `assets/`: Static resource directory.
- `assets/icon/icon.png`: App icon source file (used by `flutter_launcher_icons`).
- `test/widget_test.dart`: Flutter test example.
- `android/`: Android native project and Gradle configuration.
- `ios/`: iOS native project (including `Podfile` and Xcode project).
- `macos/`: macOS native project.
- `windows/`, `linux/`: Desktop native projects.
- `web/`: Web entry and static assets.
- `pubspec.yaml`: Dependencies, assets, versioning, and build configuration.
- `analysis_options.yaml`: Dart/Flutter static analysis rules.

## 3. Common Development Commands

Install dependencies:

```bash
flutter pub get
```

List available devices:

```bash
flutter devices
```

Run locally for debugging:

```bash
# Auto-select a device
flutter run

# Specify a device (examples)
flutter run -d ios
flutter run -d android
flutter run -d macos
flutter run -d chrome
```

Static analysis and tests:

```bash
flutter analyze
flutter test
```

## 4. Icon and Asset Maintenance

The project is configured with `flutter_launcher_icons`. After replacing `assets/icon/icon.png`, run:

```bash
dart run flutter_launcher_icons
```

If you modify asset directories or files, remember to update the `flutter` section in `pubspec.yaml` and run `flutter pub get` again.

## 5. Multi-Platform Build Commands

Android:

```bash
# Build an optimized APK (recommended for testing)
flutter build apk --release --target-platform android-arm64 --obfuscate --split-debug-info=build/symbols --tree-shake-icons

# Build an App Bundle (recommended for publishing to Google Play)
flutter build appbundle --release --obfuscate --split-debug-info=build/symbols --tree-shake-icons
```

iOS (requires macOS + Xcode environment):

```bash
flutter build ios
```

macOS:

```bash
flutter build macos
```

Web:

```bash
flutter build web
```

Windows / Linux:

```bash
flutter build windows
flutter build linux
```

## 6. Development Recommendations

- Run `flutter analyze` and `flutter test` at least once before committing.
- When changing platform-specific configuration, prioritize edits in the corresponding directories: Android in `android/`, Apple platforms in `ios/` and `macos/`.
- Since business logic is currently concentrated in `lib/main.dart`, consider splitting code into modular files under `lib/` (such as `models/`, `pages/`, and `widgets/`) to improve maintainability.
